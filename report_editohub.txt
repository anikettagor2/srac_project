## 1. Overview
EditoHub is a premium video editing agency platform built with Next.js, React, TypeScript, Tailwind CSS, Framer Motion, and Firebase. It provides a workflow for uploading raw footage, generating review-ready preview assets, and delivering fast browser playback using a modern video engineering stack. The backend uses Firebase (Firestore, Storage, Cloud Functions) and Google Cloud Transcoder for video processing.

## 4. Component Connections & Data Flow

- **Home Page**: Composes UI from `components/`, uses context providers for branding and authentication.
- **Navbar/Footer**: Use branding context for logo, auth context for user state.
- **Video Upload**: Frontend triggers upload via `UploadService`, which routes to Mux or Firebase based on file type and project date. Updates Firestore with asset URLs.
- **Review Pages**: Fetch revision/project data from Firestore, render video player and comments. Real-time updates via Firestore listeners.
- **Dashboard**: Renders different dashboards based on user role, fetches data from Firestore, uses types from `schema.ts`.

## 5. Key Workflows

### Authentication
- Managed via Firebase Auth, with context in `auth-context.tsx`.

### Video Upload & Processing
- User uploads video via frontend (UI in `components/`).
- Upload triggers logic in `UploadService`, which routes to Mux or Firebase.
- Transcoded assets are stored in Firebase Storage.
- Firestore is updated with asset URLs.

### Review & Playback
- Frontend loads preview assets using custom video player components.
- Reviewers can comment, request changes, or approve.

### Project Management
- Projects, users, revisions, comments, and invoices are all typed in `schema.ts`.
- Managed via Firestore and surfaced in the dashboard.

### Payments
- Razorpay integration for payments.
- Invoice generation and tracking.
